The problem

Every lending protocol on Ethereum today has the same architectural choice baked in from day one: transparency. Aave, Compound, Morpho — every position, every wallet, every pending liquidation is readable by anyone with a browser. That was a reasonable tradeoff in 2020. In 2026, it is a $500M/year extraction machine.

What Walnut is

Walnut is a confidential lending and borrowing protocol where every sensitive variable lives and computes as encrypted state. Deposits, debt, collateral factors, health factors, interest rates, liquidation thresholds — all stored as euint64 ciphertexts. The blockchain sees encrypted values. Only the authorized wallet sees plaintext. The protocol still runs — it just runs blind.

The name is intentional. A walnut has a hard outer shell and an encrypted kernel inside. Nobody sees the kernel until you crack it. Wallet + Nut. Encrypted at the core.

Core features — why each one needs FHE

Encrypted health factor engine
Health factor = collateral/debt. Walnut evaluates this comparison entirely in FHE. Nobody knows you're at 1.05 until the protocol decides to act. MEV bots can't target you.
FHE.div() + FHE.lt() on euint64
Sealed-bid liquidation
Liquidators submit encrypted bids. CoFHE picks the minimum penalty bid via on-chain comparison — without any bid ever being visible. Borrower loses the least. For the first time, liquidation protects the borrower.
publish-decrypt callback pattern
Multi-wallet ENS collateral
Users aggregate collateral across wallets linked to an ENS name. CoFHE sums encrypted balances — unlinking wallets on-chain. Higher borrowing power, zero wallet exposure.
FHE.add() across wallets + ENS
Encrypted credit scoring
Protocol computes a credit tier from encrypted repayment history. Higher tier = lower collateral requirement. Under-collateralized lending unlocked for proven participants. Raw data never exposed.
euint counter history + tier unlock
P2P encrypted lending
Lenders post encrypted offers: APR, tenor, size — all in ciphertext. Borrowers match without seeing terms until settlement. True private loan negotiation.
FHERC20 + Privara SDK
Selective institutional disclosure
Auditors receive a time-bounded decryption permit for aggregate pool solvency — not individual positions. Compliance without exposure. Fhenix's ACL system.
FHE.allow() + permit system

The "cannot exist on transparent rails" test

Fhenix evaluates submissions by one standard above all others: does this protocol require FHE, or is privacy cosmetic? Walnut passes this test three times over.

Sealed-bid liquidation is fundamentally broken on transparent rails. If bids are visible, liquidators simply outbid each other by 1 wei, and the borrower always pays maximum penalty. The mechanism only works because bids are encrypted.

Health factor computation requires operating on encrypted collateral and debt values simultaneously. If either is ever plaintext, front-runners can calculate your liquidation point before you can react.

Credit scoring from position history requires the scoring inputs to stay encrypted — otherwise the privacy you're trying to prove is destroyed in the act of proving it.
